1. Problem Statement¶

  • Build a regression model to predict the sales price for each house in Ames, Iowa
    • Output Variable: The Predicted SalePrice of a house

1.1 About The Data¶

  • There are 79 variables which is a lot.
    • Should find out which variable have the strongest correlation (predictive power) with SalePrice and build a model based on them
    • Beaware of variables that are correlated with each other -> Should include just 1 of them

1.2 Lessons Learnt From Titanic¶

  • Spent too much time on feature enginerring and not aware of how each step taken affects the final score.
  • More often than not, it does little to improve the accuracy, or even negatively impact the score.

6. Feature Engineering¶

  • In this section, we are going to explore log transformation on numeric features, encode ordinal categorical features, and finally encode the nominal categorical features

6.1 Log Transformation¶

  • In Section 2.1, we have already discovered that SalePrice is not normally distributed and it is skewed positively
  • In Machine Learning, many models such as Linear, Lasso, Ridge Regression models assume the input data is normally distributed
    • Note: For tree-based models, data does not need to be normally distributed
  • If we do not normalize our data, the prediction made by such models will not be accurate as the probability of finding most occurance will not be at the mean but higher or lower than the mean
  • One way to normalize data is to take the log() of the data
    • Normalization also reduces the effect of outliers in this case, rendering a more linear relationship between the features and SalePrice
  • Reference: https://stats.stackexchange.com/questions/298/in-linear-regression-when-is-it-appropriate-to-use-the-log-of-an-independent-va

7. Modelling¶

  • Linear Regression is a model that tries to model a linear relationship between the dependent variable (SalePrice) and one or more independent variables (features)
    • Only 1 independent variable - Simple Linear Regression
    • More than 1 indepedent variable - Multiple Linear Regression
  • Equation of Multiple Linear Regression: y(x) = b + b1(x1) + b2(x2) + b3(x3) + ...
    • x1, x2 ... are different features
    • b, b1, ... are coeff of features
  • From the linear regression equation above, the coeff of each feature determines how much that feature contributes to the final value of target
    • The coeffs are calculated by the linear regression model to best fit the train data; which kinds of indicate the importance of the feature
  • Reference: https://www.coursera.org/lecture/machine-learning/features-and-polynomial-regression-Rqgfz
  • After thorough research, I came across Regularization techniques in Linear Regression
    • Regularzation is often useful when you have a large number of features in the dataset. Especially for Ames Dataset.
    • It can effectively prevent over-fitting by introducing penalize terms to features, so to reduce any extreme weights (coeffs of features) and hence stop any feature from dominating
    • Another good thing is that the models which use regularization can automatically select features which help to predict the target
  • There are 2 levels of Regularization:
    • L1 Regularization, known as Lasso Regression
    • L2 Regularization, known as Ridge Regression
  • They differ by penalizing terms introduced to the features in the Linear Regression Equation
    • Lasso' penalizing terms take the sum of absolute values of feature coeffs, shrinking the less important features's coeff to 0, removing them from the model - Feature Selection
    • Ridge's penalizing terms, on the other hand, take the square sum of the coeffs of the features, and as a result, Ridge often uses all the terms
  • Reference: https://towardsdatascience.com/l1-and-l2-regularization-methods-ce25e7fc831c

scorer = "neg_mean_squared_error"

def rmsle_cv(model):
    kf = KFold(n_folds, shuffle=True, random_state=33)
    rmse= np.sqrt(-cross_val_score(model, X, y, scoring=scorer, cv=kf))
    return(rmse)
